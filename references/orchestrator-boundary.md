# Orchestrator Boundary

The future `run_hunt.py` should be glue, not a replacement agent.

## Scripts Own

- Create run directories.
- Read and validate `brief.json`.
- Build candidates from existing `raw-search.jsonl` generated by Hermes-native search.
- Run optional Serper Maps calls for Maps/local query lanes only.
- Run Jina reads.
- Extract email, phone, social, and contact-page signals.
- Deduplicate domains, emails, and company keys.
- Export JSONL and CSV.
- Compute simple statistics.

## Hermes Owns

- Convert product context into ICP.
- Choose search lanes.
- Generate search plans and local-language queries.
- Decide which website pages to inspect and when to use Jina vs Browser/Camofox.
- Judge whether a company is a plausible buyer/channel.
- Distinguish competitor manufacturer from reseller/distributor.
- Write evidence claims.
- Decide high/medium/low/reject when context is ambiguous.
- Generate next-round strategy.
- Write the run report.

## Allowed Script Outputs

Scripts may emit helper fields:

```json
{
  "rule_contactability_score": 0.72,
  "rule_source_quality": "official",
  "needs_review": true
}
```

Hermes must still make final business judgments.

## CLI Shape for Later

```bash
python scripts/run_hunt.py brief.json --out lead-runs/example --pilot --queries maps-queries.jsonl
python scripts/run_hunt.py --out lead-runs/example --build-candidates
python scripts/run_hunt.py --out lead-runs/example --enrich
python scripts/run_hunt.py --out lead-runs/example --export
```

Do not implement unbounded autonomous loops in the orchestrator.
